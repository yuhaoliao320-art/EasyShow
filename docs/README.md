# 建材行產品目錄網站 - MVP User Story 文件

## 一、MVP 專案願景

為建材行建立一個**最簡可行產品（MVP）** 目錄展示網站，核心目標是讓管理者能夠透過**無限層級的分類**（類似資料夾概念）組織產品，並在前台展示**產品名稱與圖片**。先求「能用、好用」，後續再逐步擴充規格欄位、搜尋篩選等進階功能。

---

## 二、MVP 範圍界定

| 納入 MVP                     | 暫不納入 MVP                       |
| ---------------------------- | ---------------------------------- |
| ✅ 產品名稱 + 圖片展示        | ❌ 產品規格欄位（尺寸/材質/顏色等） |
| ✅ 無限層級分類（資料夾概念） | ❌ 複雜篩選與搜尋                   |
| ✅ 後台上傳產品照片           | ❌ 批次匯入 Excel                   |
| ✅ 後台管理無限層級分類       | ❌ 庫存管理                         |
| ✅ LINE 瀏覽適配              | ❌ LIFF 進階整合                    |
| ✅ 前台瀏覽分類樹             | ❌ 詢價表單                         |
| ✅ 產品詳情頁（名稱+圖片）    | ❌ 會員/我的最愛                    |

---

## 三、目標用戶（MVP 階段）

## 五、User Stories

### Epic 1：前台瀏覽

**US-01｜瀏覽分類樹（無限層級）**
> 作為一名**顧客**，我希望能在前台看到多層級的分類結構（例如：瓷磚 → 進口瓷磚 → 義大利系列），以便像瀏覽資料夾一樣逐步深入找到我需要的產品。

- **Given** 我進入網站首頁
- **When** 頁面載入完成
- **Then** 我應該看到**第一層分類**的清單
- **And** 如果某個分類下有子分類，旁邊應顯示可展開的圖示（如 `>` 或 `＋`）
- **And** 點擊分類名稱可進入該分類的產品列表
- **And** 點擊展開圖示可顯示下一層子分類
- **And** 分類路徑應有「麵包屑」導航（如：首頁 > 瓷磚 > 進口瓷磚 > 義大利系列）

---

**US-02｜瀏覽產品列表（名稱 + 圖片）**
> 作為一名**顧客**，當我進入某個分類時，我希望看到該分類下所有產品的名稱與圖片，以便直觀地認識有哪些產品。

- **Given** 我點擊進入某個分類
- **When** 產品列表載入完成
- **Then** 我應看到產品以**卡片模式**陳列
- **And** 每張卡片至少顯示：**產品圖片** + **產品名稱**
- **And** 點擊卡片可進入產品詳情頁

---

**US-03｜查看產品詳情（名稱 + 圖片放大）**
> 作為一名**顧客**，我希望點擊產品後能進入詳情頁，看到更大的產品名稱與圖片，以便仔細查看產品外觀。

- **Given** 我在產品列表頁
- **When** 點擊某個產品卡片
- **Then** 我應進入產品詳情頁，看到：
  - 產品名稱（大字顯示）
  - 產品圖片（以較大尺寸展示）
  - 若有多張圖片，可左右滑動或點擊切換查看
  - 點擊圖片可放大檢視細節
- **And** 頁面應有「返回列表」的按鈕或連結

---

**US-04｜空狀態與提示**
> 作為一名**顧客**，當某個分類下還沒有任何產品時，我應該看到清楚的提示，而不是空白頁面，以便知道該分類目前尚無產品。

- **Given** 我進入一個尚未有產品的分類
- **When** 頁面載入完成
- **Then** 我應看到：「此分類目前尚無產品，請參考其他分類」的友善提示
- **And** 頁面應推薦顯示同層級的其他分類供顧客瀏覽

---

### Epic 2：後台分類管理（無限層級）

**US-05｜建立與管理無限層級分類**
> 作為一名**建材行管理者**，我希望能在後台建立多層級的分類結構（無限深度），就像在電腦上建立資料夾一樣，以便有彈性地組織各種建材產品。

- **Given** 我已登入後台
- **When** 進入「分類管理」功能
- **Then** 我應看到現有的分類以**樹狀結構**顯示（類似檔案總管）
- **And** 我可以：
  - **新增頂層分類**：點擊「新增分類」，填寫名稱後建立
  - **新增子分類**：在任一現有分類下點擊「新增子分類」，建立下一層級
  - **編輯分類**：修改分類名稱
  - **刪除分類**：刪除分類時需確認，若該分類下**已有產品或子分類**，必須提示無法刪除，或要求先搬移/清空
  - **移動分類**：透過拖曳或下拉選單，將整個分類（含其下所有子分類與產品）移動到另一個分類下
  - **調整排序**：在同層級分類中調整顯示順序

---

**US-06｜分類路徑顯示與管理輔助**
> 作為一名**建材行管理者**，我希望在後台管理分類時能清楚看到每個分類的完整路徑（例如：瓷磚 > 進口瓷磚 > 義大利系列），以便確認分類的層級位置是否正確。

- **Given** 我在後台「分類管理」頁面
- **When** 查看分類列表
- **Then** 每個分類應顯示其完整路徑（麵包屑格式）
- **And** 分類名稱前應以縮排或樹狀線條顯示其層級深度
- **And** 可一鍵展開/收合所有子分類，方便管理大量分類

---

### Epic 3：後台產品管理

**US-07｜新增產品（名稱 + 照片上傳）**
> 作為一名**建材行管理者**，我希望能在後台為產品設定名稱並上傳照片（至少一張），以便在前台展示給顧客看。

- **Given** 我已登入後台
- **When** 進入「產品管理」→「新增產品」
- **Then** 我可以填寫以下**必要欄位**：
  - **產品名稱**（必填，限 50 字內）
  - **所屬分類**（必選，從現有分類中選擇，含多層級分類）
  - **產品照片**（至少上傳 1 張，最多 10 張）
- **And** 可填寫**選填欄位**：
  - 產品簡短描述（選填，限 100 字內）
- **And** 設定產品是否「上架」（預設為上架）
- **And** 上傳的照片應顯示預覽縮圖
- **And** 儲存後自動回到產品列表頁

---

**US-08｜編輯、排序與刪除產品**
> 作為一名**建材行管理者**，我希望能在後台管理現有產品（編輯名稱/照片、調整照片順序、刪除產品），以便隨時更新目錄內容。

- **Given** 我已登入後台，進入「產品管理」列表
- **When** 查看產品列表
- **Then** 每個產品應顯示：縮圖、名稱、所屬分類、上架狀態、建立日期
- **And** 我可以：
  - **編輯**：修改產品名稱、分類、照片（可新增/刪除/替換照片）
  - **調整照片順序**：以拖曳方式調整多張照片的顯示順序（第一張為主要圖片）
  - **切換上架狀態**：一鍵切換「上架」/「下架」
  - **刪除**：刪除產品時需二次確認
- **And** 列表支援關鍵字搜尋（依產品名稱搜尋）
- **And** 列表可依「所屬分類」過濾顯示

---

### Epic 4：LINE 瀏覽支援（MVP）

**US-09｜LINE 瀏覽器基本適配**
> 作為一名**顧客**，當我透過 LINE 開啟建材行目錄網站時，頁面應能完整顯示且操作順暢，以便在手機上輕鬆瀏覽。

- **Given** 我在 LINE 應用程式中點擊網站連結
- **When** 頁面在 LINE 瀏覽器中載入
- **Then** 頁面應：
  - 完整顯示，無內容被切掉或遮擋
  - 圖片自動縮放至適合手機螢幕寬度
  - 文字大小適中（不需手動縮放即可閱讀）
  - 所有按鈕與連結都容易用手指點擊（至少 44x44pt）
  - 分類樹的展開/收合按鈕在觸控操作下反應靈敏

## 六、使用情境（MVP）

### 情境一：建材行管理者建立分類架構

1. **建材行林店長** 決定開始使用網站展示產品，首先需要建立分類結構
2. 登入後台，進入「分類管理」
3. 先建立頂層分類：
   - 點擊「新增分類」，輸入「瓷磚」
   - 再新增「木地板」、「衛浴設備」、「油漆塗料」
4. 接著在「瓷磚」下建立子分類：
   - 點擊「瓷磚」旁的「新增子分類」，輸入「進口瓷磚」
   - 再新增「國產瓷磚」、「復古磚」
5. 繼續在「進口瓷磚」下建立更細的分類：
   - 新增「義大利系列」、「西班牙系列」、「日本系列」
6. 最終分類結構像這樣：
   ```
   ├── 瓷磚
   │   ├── 進口瓷磚
   │   │   ├── 義大利系列
   │   │   ├── 西班牙系列
   │   │   └── 日本系列
   │   ├── 國產瓷磚
   │   └── 復古磚
   ├── 木地板
   │   ├── 實木地板
   │   └── 超耐磨地板
   ├── 衛浴設備
   └── 油漆塗料
   ```
7. 所有分類建立完成後，林店長到前台確認分類樹正確顯示

### 情境二：管理者上傳產品

1. 林店長進了 10 款新的義大利瓷磚，要上架到網站
2. 登入後台，進入「產品管理」→「新增產品」
3. 輸入產品名稱：「義大利進口仿古磚 - 托斯卡納系列 A1」
4. 選擇所屬分類：瓷磚 > 進口瓷磚 > 義大利系列
5. 從電腦選取 5 張產品照片（不同角度、近拍紋理），點擊上傳
6. 確認所有照片上傳成功後，點擊「儲存」
7. 重複步驟 3-6，完成 10 款產品的上架
8. 到前台點擊「瓷磚 > 進口瓷磚 > 義大利系列」，確認 10 款產品都有顯示

### 情境三：顧客在 LINE 中瀏覽目錄

1. **設計師小吳** 在 LINE 群組中看到朋友分享的建材行目錄連結
2. 點擊連結，網站在 LINE 應用程式內直接開啟
3. 首頁看到分類樹，展開「瓷磚」看到「進口瓷磚」和「國產瓷磚」
4. 繼續展開「進口瓷磚」看到「義大利系列」、「西班牙系列」、「日本系列」
5. 點擊「義大利系列」，看到該分類下所有產品的卡片列表（圖片+名稱）
6. 對「托斯卡納系列 A1」感興趣，點擊進入詳情頁
7. 看到大尺寸的產品名稱與多張產品照片
8. 點擊照片放大查看紋理細節
9. 覺得符合客戶需求，把產品頁面截圖傳給客戶討論

### 情境四：管理者調整分類與產品

1. 林店長發現「復古磚」賣得很好，需要獨立在首頁顯示
2. 進入後台「分類管理」，原本「復古磚」在「瓷磚」下
3. 將「復古磚」**拖曳**到頂層，變成獨立分類
4. 分類結構變成：
   ```
   ├── 瓷磚
   │   ├── 進口瓷磚
   │   └── 國產瓷磚
   ├── 復古磚  ← 獨立出來了
   ├── 木地板
   └── ...
   ```
5. 到前台確認「復古磚」已出現在首層分類中
6. 接著發現一款產品分類歸錯了，進入「產品管理」編輯該產品，將所屬分類從「進口瓷磚」改為「復古磚」
7. 儲存後到前台確認產品已正確顯示在「復古磚」分類下

### 情境五：顧客瀏覽空的子分類

1. **工班師傅阿強** 點進「木地板 > 實木地板」分類
2. 但林店長還沒上傳任何實木地板產品
3. 頁面顯示：「此分類目前尚無產品，請參考其他分類」
4. 同時頁面推薦了「超耐磨地板」和「瓷磚」等其他分類
5. 阿強點擊「超耐磨地板」瀏覽，找到了需要的產品

## 七、資料結構（簡化版）

## 八、MVP 驗收標準

| 項目           | 驗收條件                                          |
| -------------- | ------------------------------------------------- |
| ✅ 分類無限層級 | 後台可建立至少 5 層以上的分類結構，前台能正確顯示 |
| ✅ 分類移動     | 後台可將分類（含子分類與產品）移動到另一個分類下  |
| ✅ 產品上傳     | 後台可新增產品名稱 + 上傳至少 1 張照片            |
| ✅ 產品列表     | 前台分類頁顯示產品卡片（圖片+名稱）               |
| ✅ 產品詳情     | 前台點擊產品顯示大圖與名稱，多張照片可切換        |
| ✅ 空狀態       | 無產品時顯示友善提示                              |
| ✅ LINE 適配    | 在 LINE 瀏覽器中頁面完整顯示、按鈕可點擊          |
| ✅ 後台權限     | 需登入才能進入後台管理功能                        |

## 九、MVP 技術建議

| 項目         | 建議                                                         |
| ------------ | ------------------------------------------------------------ |
| 圖片儲存     | 上傳時自動產生兩種尺寸：縮圖（300x300）與原圖，前台依情境使用 |
| 分類樹實作   | 使用「鄰接列表模型（Adjacency List）」搭配遞迴查詢，或使用「巢狀集合模型（Nested Set）」提高查詢效率 |
| 前台分類展開 | 建議一開始**只展開第一層**，點擊才展開子層級，避免一次載入過多資料 |
| 後端 API     | 建議提供分類樹的單一 API 回傳完整結構，前端一次載入後自行渲染 |
| LINE 適配    | 使用 `viewport` 設定，加上 `safe-area-inset` 處理瀏海與底部操作區 |

## 十、附註

- MVP 階段**不包含**規格欄位（尺寸/材質/顏色等），僅有「產品名稱」與「圖片」
- 分類的無限層級是這個 MVP 的核心亮點，讓建材行能像管理檔案資料夾一樣靈活組織產品
- 產品名稱建議限制字數，確保前台卡片排版整齊
- 第一張上傳的照片自動設為「主要圖片」，用於列表頁縮圖顯示
